Bootstrap 4 Forms

Responsive and consistently forms

Bootstrap 4 provides a default style that is applied across most form controls. It also provides classes for specific purposes.

In particular, Bootstrap applies display: block and width: 100% to most form controls. This results in the form controls being stacked vertically by default (but you can change this if required).

Basic Form

Use Bootstrpa's .form-control class against textual <input>, <textarea> and <select> elements to apply width: 100% by default.

Wrap labels and controls inside a <fieldset> with the Bootstrap .form-group class applied for optimum spacing. Alternatively, you can use a <div&glt; or other element.

Inline Form

use Bootstrap's .form-inline class to make the form elements render as inline-block and left-aligned. It's work only when the viewport width is at least 768 pixels wide.

Hidden Labels

Labels should always be provided on input fields, otherwise screenreaders will have troub. You can hide the label if required, by using Bootstrap's .sr-only class

Horizontal Form

You can use Bootstrap's .form-horizontal to align labels and form controls horizontally.

When doing this, you need to use Bootstrap's grid classes to specify how many columns each elements should span. Specifically, add a .row class to the .form-group and a .col-*-* class for each column.

Also, you can add Bootstrap's .form-control-label class to the <label> element in order to vertically center the label in relation to textual input elements.

Form Control Size

You can use Bootstrap 4's .form-control-lg and .form-control-sm to increase or decrease the size of an input control.

Column Size

You can use Bootstrap's grid system to constrain input controls to desired width.

Help Text

Bootstrap 4 provides the .text-muted class that can be used to indicate help text. This can be applied to any inline HTML element (such as the <p>, <span> or <small> elements).

You should also use the .aria-describedby attribute to associate the help text with the form control. This will ensure that screen readers can announce the help text when the user focuses on the form control

Validation Styles & Feedback Icons

Bootstrap includes validation styles for form controls. These can be used to convey one of three validation status (success, danger, or warning) to the user.

The three classes are .has-success, .has-warning and .has-danger. These should be applied to the parent element, so that any .control-label, .form-control and .text-muted element nested within that element will inherit the validation styles.

Also, textual input controls can make use of feedback icons. To do this, add a .form-control-success, .form-control-warning or .form-control-danger class to the .form-control input field.

You can use Bootstrap's .form-control-static on a <p> element to present plain text next to a label(eg. instead of using an input field).

Disabled

Bootstrap4 4 provides classes for diabled and inline checkboxs and radio buttons. To make use of these classes, nest the checkbox or radio inside a <div> with either the .radio, .radio-inline, .checkbox or checkbox-inline class as required.

Disabled

Whenerver you use the standard html disabled attribute to disable a checkbox or radio button, you also have the option of adding Bootstrap's .disabled class so that the "Not Allow" cursor appears whenever the user hovers over that control.

Inline

With Bootstrap4, checkboxes and radio buttons are stacked by default(they appear above one another).

You can display them inline by using the .radio-inline or .checkbox-inline class(depending on which element you're displaying).
